## Design Context

### Users
Non-technical rugby coaches and performance analysts using iPhone or iPad during live review or quick post-match review. They need to log ball-in-play state changes fast, trust the running clock, and export clean event data without setup friction.

### Brand Personality
Sharp, professional, approachable.
The interface should feel credible for analysis work, calm under pressure, and quick to learn on first use.

### Aesthetic Direction
Dark analytics workspace inspired by the Effort Profiling dashboard and LatchR. Use charcoal, bone, muted grass and copper accents, crisp geometry, and strong hierarchy. Avoid toy-like sports UI, neon dashboards, soft consumer-app styling, and decorative gradients that do not support the workflow.

### Design Principles
1. Primary actions must dominate the screen: play, pause, ruck, and the live clock.
2. Data should feel operationally reliable: every state, timestamp, and export cue must read clearly.
3. Color should encode workflow state, not decoration.
4. Dense information should still feel calm on iPhone and iPad through spacing and hierarchy.
5. Export and recovery must feel safe: local auto-save should be visible and trustworthy.
